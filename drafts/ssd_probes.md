- Purpose
  - Validate SSD claims that forks are preserved and locks are sharpened
- Probing mechanism
  - Carrier selection (response): either base or trained model; stratify by things like length, which model got this prompt correctly.
  - Token selection:
    - Regular intervals (stratify by early, mid and late reasoning)
    - Known tokens of interest: likely locks (formatting, \\boxed, etc.), likely forks (Let, Suppose, ...)
    - Random
  - At the selected tokens, compute and compare outputs of both the baseline and trained model
  - Types of tokens
    - Lock-like heuristic: high top-1 probability / small nucleus
    - Fork-like heuristic: multiple tokens in top-5 or 10 with similar probability, significant nucleus mass (distinguishes from diffuse)
- T=1.5 analysis
  - Fuzzy initial stats
  - Lock sharpening
    - Heuristics don't tell the whole story
    - E.g. bad Wait token --> also lines up with what we found about the model 
    - Decompose into
      - strict locks: top-1 token is the same
      - Swaps
      - Self-correction: Wait, let's, check, etc.
    - We find an average sharpening on strict locks when excluding these.
    - Overall, qualitatively we can find some examples but that doesn't translate in stats we can describe.
  - Fork preserving
    - Works, but forks are still somewhat compacted; core alternatives are still
      preserved
- Training effect vs SSD effect: hard to distinguish entirely; would need a rho=null baseline (as called out in the paper)
